# Product Agent

A service that reads vague GitHub issues and converts them into structured,
engineering-ready specifications using an LLM and local codebase context.

---

## How it works

```
GitHub Issue (webhook)
        │
        ▼
  Extract keywords
        │
        ▼
  Scan .go files in REPO_PATH → top 5 relevant files
        │
        ▼
  Build prompt (issue + code context + schema)
        │
        ▼
  Call OpenAI-compatible API
        │
        ▼
  Parse JSON → AgentOutput
        │
        ▼
  Format as Markdown → post GitHub comment
```

---

## Project structure

```
/product-agent
├── main.go        # Entry point, config, HTTP server
├── handler.go     # Webhook HTTP handler + pipeline orchestration
├── github.go      # GitHub API: parse webhook payload, post comment
├── retriever.go   # Scan .go files, keyword scoring, content truncation
├── agent.go       # OpenAI-compatible LLM client with retry
├── prompt.go      # Prompt construction with embedded JSON schema
├── types.go       # Shared data structures
└── utils.go       # Keyword extraction, Markdown formatting, helpers
```

## Configuration (environment variables)

| Variable         | Required | Default                        | Description                        |
|------------------|----------|--------------------------------|------------------------------------|
| `GITHUB_TOKEN`   | ✅        | —                              | GitHub personal access token       |
| `OPENAI_API_KEY` | ✅        | —                              | OpenAI (or compatible) API key     |
| `OPENAI_BASE_URL`| ❌        | `https://api.openai.com/v1`   | Base URL for OpenAI-compatible API |
| `OPENAI_MODEL`   | ❌        | `gpt-4o`                       | Model name                         |
| `REPO_PATH`      | ❌        | `.`                            | Absolute path to local repository  |
| `PORT`           | ❌        | `8080`                         | HTTP server port                   |

---

## Run the server

```bash
export GITHUB_TOKEN=ghp_your_token
export OPENAI_API_KEY=sk-your_key
export REPO_PATH=/path/to/your/repo
export OPENAI_MODEL=gpt-4o       # optional

go build -o product-agent .
./product-agent
```

## Test locally with curl

```bash
curl -X POST http://localhost:8080/webhook \
  -H "Content-Type: application/json" \
  -H "X-GitHub-Event: issues" \
  -d @testdata/issue_opened.json
```

### ⚠️ Edge Cases
- Clients behind a shared NAT/proxy share the same IP — document this limitation
- Clock skew affecting sliding window calculations
